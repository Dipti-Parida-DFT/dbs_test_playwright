13. Reporting (Dual Reporter Strategy)

The framework employs a dual-reporter strategy — Playwright HTML for real-time debugging and Allure for historical trend analysis and stakeholder reporting.

Playwright HTML Reporter

AspectDetail
Output directoryplaywright-report/
Auto-open behaviourOpens in browser automatically when a test fails
Execution commandnpx playwright show-report

Capabilities

  • Step-by-step trace view: Replay each test action with DOM snapshots, network logs, and console output
  • Screenshot on failure: Automatic full-page screenshot captured at the point of failure
  • Video playback: Retained video recordings for failed tests (configurable via retain-on-failure)
  • Trace viewer: Interactive timeline with before/after DOM state for each action — available on first retry
  • Filtering & search: Filter by test status (passed, failed, skipped, flaky), search by test name
  • Attachment support: Inline display of screenshots, PDFs, and text attachments within the report
  • Shareable: Self-contained static HTML — can be distributed via email or hosted on any web server

Allure Reporter

AspectDetail
Raw resultsallure-results/ (JSON result files + attachment binaries)
Generated reportallure-report/ (static HTML dashboard)
Generate commandnpm run allure:generate → allure generate allure-results --clean -o allure-report
Open commandnpm run allure:open → allure open allure-report
Combined shortcutnpm run test:report (generate + open in one command)

Capabilities

  • Dashboard overview: At-a-glance pass/fail/broken/skipped distribution with pie charts and severity breakdown
  • Trend charts: Historical pass/fail trends across multiple runs — tracks regression patterns over time
  • Test categorisation: Tests grouped by suite, feature, story, severity, and custom labels
  • Environment metadata: Displays environment details (browser, OS, URL) alongside results
  • Timeline view: Visualises test execution order and duration on a Gantt-style timeline
  • Defect classification: Automatically categorises failures into product defects vs. test defects
  • Attachment integration: Screenshots, trace files, and text logs linked directly to each test step
  • Retries & flaky detection: Distinguishes flaky tests from genuine failures across retry attempts
  • Exportable: Report is fully static HTML — can be archived, shared, or deployed to CI dashboards

15. AI Migration Governance

Three governance documents in tests/testContexts/ guide the Protractor → Playwright migration.

context.md — Framework Standards & Analysis Phase

Defines the "Automation Test Migration Specialist" role and mandates a 5-phase analysis:

  1. Identify test metadata
  2. Extract execution steps
  3. Catalog validations
  4. Map data dependencies
  5. Document element locators

Also covers: conversion rules, checklist template, output specs, exception handling, success criteria, and quality gates.

learnings.md — Runtime Observations

Accumulated best practices discovered during live test execution:

  • Authentication handling — call handleAuthIfPresent() only on first navigation
  • UX loading — only invoke where spinners actually appear
  • Timeout strategy — 15 min for multi-phase, 5 min for single-phase tests
  • Transaction reference capture — try-catch with fallback search
  • Checkbox toggle — label for="id" click + verify/retry (3 attempts)
  • ShuRu locators — use standard Playwright selectors, not shu-ru prefix
  • Email fields — getByRole('textbox', {name:'Email'}).nth(N)
  • Payee Nickname — mandatory field, in-memory locator
  • View page locators — individual id-based selectors

migration-execution-prompt.md — 9-Step Workflow

StepAction
1Inspect — Per-page DOM analysis before interaction
2Validate — Locator & data verification against live DOM
3Continue — Execute following framework standards
4Generate Playwright spec file
5Execute & iterate until green
6Validate assertions match Protractor source
7aDocument migration summary
7bUpdate learnings.md with new findings
8Generate HTML migration chat log
9Final quality-gate review
